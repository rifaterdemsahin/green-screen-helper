# Green Screen Helper Workflow

This document outlines the development and deployment process for the Green Screen Helper project.

## Deployment

-   **Source Control:** The project is hosted on GitHub.
-   **CI/CD:** We use GitHub Actions for continuous integration and deployment. The workflow is defined in `.github/workflows/static.yml`.
-   **Structure:** This project follows the Delivery Pilot structure.

## Project Structure

The project is organized based on the following conceptual layers:

-   **Imaginary:** High-level concepts and ideas.
-   **Index:** The entry point and main structure of the application.
-   **Symbols:** Representations and UI elements.
-   **Environment:** Configuration and environment-specific settings.
-   **Formula:** Business logic and core algorithms.
-   **Real:** The tangible, running application.
-   **Semblance:** The user-facing appearance and experience.
-   **Testing:** Quality assurance and verification.

This structure helps to separate concerns and organize the codebase in a logical manner.
